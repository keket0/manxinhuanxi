# AGENT_RULES_CHECKLIST.md

更新时间：2026-04-20

这个清单用于新建、安装、巡检各个 agent，确保都继承主人的统一长期规则。

## 必须继承的总规则

1. **长期记忆**
   - 具备可持续维护的长期记忆文件或机制
   - 重要稳定规则要能沉淀，不只停留在临时上下文

2. **记忆检索**
   - 回答过去工作、偏好、决定、日期、待办前，先做记忆检索
   - 不凭印象回答历史信息

3. **每日总结与日记**
   - 每天做过的任务要定期总结
   - 至少保留 daily note / 工作记录
   - 建议包含：今日完成、关键决定、遇到的失败/阻塞、下一步

4. **技能补充路线**
   - 缺能力时，优先从官方技能站补
   - 其次从 GitHub 高 star 免费来源补
   - 前提必须免费
   - 安装后先做最小可用验证，再纳入长期工具集

## 推荐落地位置

### AGENTS.md
应至少体现：
- 使用长期记忆
- 不把重要规则只留在会话里
- 需要写入日记或工作记录

### USER.md
应至少体现：
- 主人的长期偏好
- 记忆检索要求
- 每日总结与日记要求
- 免费技能优先策略

### HEARTBEAT.md
应至少体现：
- heartbeat 时优先推进当前工作
- 定期补 daily note / 工作记录
- 回答历史事项前先检索记忆
- 缺能力时按“官方技能站 -> GitHub 高 star 免费来源”补能力

### memory/ 或工作记录目录
应至少具备：
- 可持续追加的 daily note
- 能看出最近任务是否被记录

## 快速巡检项

- [ ] 有 AGENTS.md / USER.md / HEARTBEAT.md
- [ ] 明确写了“先记忆检索再回答历史事项”
- [ ] 明确写了“每天要总结并写日记/工作记录”
- [ ] 明确写了“缺能力优先官方技能站或 GitHub 高 star 免费技能”
- [ ] 已存在 memory 目录或等价工作记录目录
- [ ] 最近有实际记录，而不是空壳文件

## 结论标准

- **通过**：规则已写明，且有实际 daily note / 记录在运行
- **部分通过**：规则已写明，但某些支撑文件缺失或记录不稳定
- **不通过**：规则未写明，或没有实际记忆/日记落地
